# Bysel (Бусел) v4.0: Sovereign Any-Scale 1-bit Omni-LLM

Bysel (Бусел) — сверхэффективная, безтокенная Any-to-Text 1-битная LLM с гибридным линейным вниманием, динамической глубиной и смесью экспертов, разработанная для работы как на высокопроизводительных серверах (NVIDIA CUDA), так и на потребительском оборудовании (Apple Silicon / CPU).

Проект спроектирован по строжайшим стандартам отказоустойчивости и оптимизации вычислений, минимизируя накладные расходы на диспетчеризацию ядер GPU и пропускную способность видеопамяти.

---

## 🚀 Архитектурные особенности

### 1. Тернарные веса (BitNet 1.58b)
Линейные слои бэкбона используют 1.58-битную тернарную математику квантования весов в диапазон `{-1.0, 0.0, 1.0}` с динамическим поканальным масштабированием активаций (INT4/INT8). Это позволяет имитировать низкобитные вычисления в процессе обучения и делает модель готовой к нативной компиляции в сверхбыстрые аддитивные ядра (без умножений).

### 2. Гибридный токен-миксер (3:1 GDN-2 & MLA)
Для обеспечения баланса между бесконечной длиной контекста и точностью извлечения фактов модель использует пропорцию слоев 3:1:
*   **GDN-2 (Gated DeltaNet-2):** 75% слоев работают на линейном внимании от NVIDIA Research (май 2026). Архитектура использует раздельные поканальные гейты стирания и записи (Decoupled Erase and Write), что решает проблему забывания ассоциаций и гарантирует константный размер кэша контекста ($O(1)$) независимо от длины последовательности.
*   **MLA (Multi-Head Latent Attention):** 25% слоев выполняют функцию глобального внимания с латентным сжатием векторов ключей и значений до размера $d_c = 128$. Это гарантирует точную работу ассоциативной памяти на контекстах до 1 000 000 токенов при минимальном расходе VRAM.

### 3. Смесь экспертов (MoE) с Blackboard Memory
Блок MoE содержит 2 постоянно активных общих эксперта (Shared Experts) и 64 разреженных эксперта (Routed Experts), выбираемых по алгоритму Top-2. Эксперты обмениваются информацией через централизованную шину памяти (Blackboard Memory), предотвращая коллапс репрезентаций и обеспечивая естественную специализацию экспертов под разные модальности данных.

### 4. Оптимизатор Moonlight-Muon с Transpose-Stabilizer
Ортонормальный градиентный оптимизатор Muon дополнен математической стабилизацией **Transpose Trick**: при работе с высокими матрицами (например, весами MoE) алгоритм Ньютона-Шульца автоматически транспонирует матрицу, исключая появление нулевых собственных значений, приводящих к `NaN`, и ускоряет шаг оптимизации в 4 раза.

### 5. Multi-Token Prediction (MTP-4)
Модель содержит 4 параллельные предсказательные головы, способные прогнозировать 4 последующих байта за один шаг, что значительно повышает скорость авторегрессионного инференса и качество долгосрочного планирования.

## 🛠 Кроссплатформенная оптимизация

Движок Bysel спроектирован кроссплатформенным и обеспечивает максимальную утилизацию любого доступного железа:

*   **На CUDA (NVIDIA):** 
    *   Автоматически задействуются быстрые Triton-ядра `flash-linear-attention` для расчета GDN-2.
    *   Шаг Muon выполняется на скомпилированном Triton-оптимизаторе `flash-muon` прямо на Tensor Cores.
    *   Включается монолитная компиляция графа всей модели `torch.compile(mode="max-autotune")`.
*   **На MPS (Apple Silicon Mac):** 
    *   512-шаговый цикл GDN-2 свернут в сверхбыстрый TorchScript JIT-граф `stable_gdn2_recurrent_jit`, минимизирующий нагрузку на шину объединенной памяти Mac в 4000 раз.
    *   Автокаст MPS переведен на нативный, аппаратно ускоренный тип `float16`.
    *   Многопроцессорный загрузчик переключается в однопоточный режим `num_workers=0` для обхода ошибок `spawn` в macOS.

## 📂 Структура репозитория

```text
bysel/
├── configs/
│   └── default.yaml     # Откалиброванные профили Ziaziulia (360M) и Zubr (24B)
├── model/
│   ├── patching.py      # Strided FastBLT байтовый патчер с F.embedding фиксом
│   ├── layers.py        # BitLinear, кастомный RMSNorm и слитные Gate-Up GLU-блоки
│   ├── attention.py     # Стабильный JIT GDN-2 и MLA со сжатием кэша
│   ├── routing.py       # MoD-роутер и разреженный MoE с Blackboard шиной
│   └── backbone.py      # Оркестратор ByselModel (3:1 Hybrid) и MTP-4 головы
├── data/
│   └── pipeline.py      # Потоковый Byte-Streamer на Rust с безопасным CPU int32-колейтором
├── training/
│   ├── optimizer.py     # Moonlight-Muon с Transpose Trick + AdamW
│   ├── autopilot.py     # Автотюнер параметров и планировщик LR
│   └── recipe.py        # Стабильные Native-целевые функции потерь
├── tests/
│   └── profiler_run.py  # Высокоточный, защищенный от зависаний профайлер шага v2.0
├── Cargo.toml           # Сборщик Rust-расширения для nvme-io
├── src/
│   └── lib.rs           # Многопоточный Safe-Streamer на Rust (mmap-based)
├── train.py             # Главный кроссплатформенный оркестратор обучения
└── debug_nan.py         # Изолированный диагностический стенд перехвата аномалий
```

## 💻 Быстрый старт

### 1. Подготовка окружения
Рекомендуется использовать современный пакетный менеджер `uv`:

```bash
# Инициализация окружения и установка зависимостей
uv venv
uv pip install -r bysel.egg-info/requires.txt
```

### 2. Сборка модуля Rust I/O
Для компиляции сверхбыстрого загрузчика байтов на Rust выполните сборку расширения:

```bash
# Сборка и интеграция расширения в Python
uv pip install -e .
# или через maturin напрямую
uv run maturin develop
```

### 3. Локальное профилирование
Перед запуском обучения рекомендуется выполнить диагностический прогон профайлера для проверки стабильности и замера производительности вашего железа:

```bash
uv run tests/profiler_run.py
```

### 4. Запуск обучения
Для запуска процесса претрейна на доступном ускорителе (CUDA или MPS выберутся автоматически):

```bash
# Запуск профиля Зязюля (360M)
uv run train.py --profile ziaziulia

# Запуск профиля Зубр (24B) на CUDA-серверах
uv run train.py --profile zubr
```
